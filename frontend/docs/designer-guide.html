1. About This Platform

What is CosmicForge?

CosmicForge is a video meeting platform similar to Google Meet and Zoom. Users can create video meetings, invite others, and communicate through video, audio, and chat.

Design Reference

Look at Google Meet and Zoom for inspiration on layout and user experience. CosmicForge should feel modern, clean, and professional - suitable for business use.

Key Features

  • Create Meetings - Schedule meetings with a title and time
  • Join Meetings - Join using a meeting code or link
  • Video & Audio - Turn camera and microphone on/off
  • Screen Sharing - Share your screen with others
  • Chat - Send text messages during meetings
  • Waiting Room - For private meetings, host approves who joins
  • Participant Management - Host can mute or remove participants

Design Freedom

This guide describes what each screen needs and what each input does. You have full creative freedom for:

  • Color palette and theme
  • Layout and spacing
  • Typography choices
  • Icons and illustrations
  • Animation and transitions
Keep It Simple

Video meetings should be easy. Users should be able to create or join a meeting in seconds. Don't overcomplicate the interface.

2. User Types

Who Uses This Platform?

Host (Meeting Creator)

The person who creates the meeting. They have full control:

  • Can start and end the meeting
  • Can admit or deny people from waiting room
  • Can remove (kick) participants
  • Can mute other participants
Participant (Invited Person)

Someone who joins a meeting created by a host. They can:

  • Turn their own camera/microphone on or off
  • Share their screen (if allowed)
  • Send chat messages
  • Leave the meeting anytime
Guest (No Account)

Someone who joins without creating an account. They have the same abilities as a participant but don't need to sign up. They just enter their name when joining.

Meeting Types

Type How It Works
Public Meeting Anyone with the meeting code can join immediately
Private Meeting People must wait in a "waiting room" until the host admits them

3. Landing Page

1 Landing Page (Homepage)
The first page visitors see. Should convince them to sign up or join a meeting.
Elements Needed
Logo CosmicForge brand logo at the top
Navigation Menu Links: Features, Pricing (optional), Login, Sign Up
Hero Section Big headline, short description, main action buttons
Quick Join Box Input field for meeting code + "Join" button (like Google Meet)
Feature Highlights 3-4 cards showing main features with icons
Call to Action "Start Free" or "Sign Up" button - prominent
Footer Links, copyright, social media icons
Meeting Code Input (Quick Join)
Allows visitors to join a meeting without signing up
Placeholder: "Enter meeting code" Example: ABCD1234 8 characters
Look at Google Meet

Google Meet's homepage has a clean split: left side shows info/actions, right side shows an illustration. There's a prominent "New meeting" button and a code input field.

4. Sign Up & Login Screens

2 Sign Up Page
New users create an account here.
Username
A unique name to identify the user on the platform
Placeholder: "Choose a username" Required Letters, numbers, underscores only
Email Address
User's email for login and notifications
Placeholder: "you@example.com" Required Must be valid email format
Password
Secret password for account security

Other Elements

Terms Checkbox "I agree to the Terms of Service"
Login Link "Already have an account? Log in"
3 Login Page
Existing users sign into their account.
Email Address
The email they used to sign up
Placeholder: "you@example.com" Required
Password
Their account password
Placeholder: "Enter your password" Required Show/Hide toggle icon
Other Elements
Login Button Primary button to sign in
Google Login Button: "Continue with Google"
Forgot Password Link "Forgot your password?" - leads to reset
Sign Up Link "Don't have an account? Sign up"
Design Tip

Keep login and signup forms centered on the page. Use a card/box container. Show only essential fields - don't overwhelm users.

5. Email Verification & Password Reset

4 Email Verification Page
After signing up, users must verify their email by entering a 6-digit code sent to their inbox.
6-Digit Verification Code
A code sent to the user's email to confirm they own it
6 separate input boxes (one per digit) Numbers only Auto-focus next box after typing
Other Elements
Instruction Text "We sent a code to [email]. Enter it below."
Verify Button Submit the code
Resend Link "Didn't receive it? Resend code"
Visual Reference

The 6-digit code input should look like 6 small boxes in a row: [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ] [ 6 ]. When user types, cursor automatically moves to next box. Similar to 2FA inputs on banking apps.

5 Forgot Password Page
User requests a password reset link/code.
Email Address
The email linked to their account
Placeholder: "Enter your email" Required
Other Elements
Instruction Text "Enter your email and we'll send a reset code"
Send Code Button Primary action button
Back to Login Link Return to login page
6 Reset Password Page
User enters the code from email and creates a new password.
6-Digit Reset Code
Code received via email
6 separate input boxes Numbers only
New Password
The new password they want to use
Placeholder: "Enter new password" Minimum 8 characters Show/Hide toggle
Confirm New Password
Type it again to confirm
Placeholder: "Confirm new password" Must match above
Other Elements
Reset Password Button Submit and change password
Success Message "Password changed! You can now log in."
* * *

Error Messages to Design

Design how these error states look (red text, icons, etc.):

  • "Invalid email address"
  • "Password must be at least 8 characters"
  • "Passwords do not match"
  • "Invalid code. Please try again."
  • "Email already registered"
  • "Incorrect email or password"

8. Join Meeting

9 Join Meeting Page
Where users enter a meeting code to join. Can also be accessed via direct link.
Meeting Code
The 8-character code shared by the host
Placeholder: "Enter meeting code" Example: ABCD1234 8 characters, letters and numbers
Your Name (for guests)
If user is not logged in, they enter a display name
Placeholder: "Enter your name" Required for guests This name shows in the meeting
Other Elements
Join Button Primary button to proceed to meeting
Meeting Info (if available) Show meeting title and host name after entering valid code
Error State "Meeting not found" if code is invalid
User Experience

When someone clicks a meeting link (e.g., cosmicforge.com/join/ABCD1234), the code should auto-fill. They just need to click "Join" or enter their name if they're a guest.

Join Flow

Enter Code → Enter Name (guests) → Pre-Join Screen → Meeting Room

10. Waiting Room

11 Waiting Room (For Participants)
When joining a private meeting, participants wait here until the host admits them.
Elements
Meeting Title Name of the meeting they're trying to join
Waiting Message "Please wait, the host will let you in soon"
Loading/Waiting Animation Subtle animation showing they're waiting
Leave Button Option to leave while waiting
12 Waiting Room Panel (For Host)
The host sees this panel inside the meeting room. Shows people waiting to be admitted.
Elements
Waiting Count Badge Number showing how many people are waiting
Waiting Participant Row Shows name of each waiting person
Admit Button Green button to let someone in
Deny Button Red button to reject someone
Admit All Button Let everyone in at once (optional)
Notification

When someone enters the waiting room, the host should see a notification or badge on the "Participants" button so they know someone is waiting.

Speaking Indicator Border glow or icon when person is talking
Host Badge Small label showing who is the host
Avatar Placeholder Shown when camera is off (initials or icon)

12. Meeting Ended Screen

17 Meeting Ended Page
Shown after leaving or when meeting ends. Simple confirmation with next actions.
Elements
Message "You left the meeting" or "The meeting has ended"
Meeting Summary (optional) Duration, participant count
Rejoin Button If meeting still active: "Rejoin meeting"
Return to Home Button "Return to dashboard" or "Go home"
Feedback (optional) "How was your meeting?" with star rating or thumbs
Keep It Simple

This screen doesn't need to be complex. A simple message and a button to go back home is enough. Don't force feedback.

Meeting States Summary

State What User Sees
Scheduled Meeting is set for the future. Shows in "Upcoming" list.
Ongoing Meeting is happening now. Can be joined.
Ended Meeting is over. Shows in "Past meetings" list.
